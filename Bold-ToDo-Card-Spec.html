Component spec

To-Do Card

The atomic unit of the Today → “To do” list. One card = one action for today. Every variant shares a single layout — icon left, heading + optional subcopy, trailing control — and swaps only the trailing control and its behavior.

Source: Bold-Wireframes.html · planItem() Figma: Member Dashboard · node 969-13205 Selector: .ci-card

1 · Where it lives

The “To do” list is pulled from the member’s care plan, authored by their provider, and updates after each visit. Here is the full section as it renders in Today.

Section chrome

  • Header — “To do” (serif 18px) + live count (2/4 done).
  • Intro sub-copy — states the source: provider-written, updates after each visit.
  • Footer note — separates the two behavioral families.

Two families

  • To-do tasks (labs, measurements) — system-verifiable; check off when actually done.
  • Daily habits (water, protein, movement) — honor-system; mark what you like or tell Bold in natural language.
Source model. Care-plan items are provider-specific. Recommended items (e.g. the fitness portion) are program-level and can deep-link to the Bold weight-management program rather than a provider directive.

2 · Anatomy

Head row is a 3-part flex: fixed icon · growing text column · pinned trailing control. Pills sit full-width below.

Parts

  • Icon — always present. 40×40 circle, blue fill + white glyph — or a photo (fitness class → trainer headshot) via img.
  • Heading — required. Verb-first, one line preferred.
  • Subcopy — optional. The “why” or provenance. Omit for compact, heading-only cards.
  • Trailing control — right-pinned; varies by variant.
  • Quick actions — optional; the card’s AI entry points (“Ask about this move”, “Easy protein ideas”) and program links. Tapping one opens Bold with that context.

3 · Variants

Same layout throughout — only the trailing control and tap behavior change. Try the interactive controls.

4 · Content variants

Orthogonal to the behavioral variants above: subcopy, pills, and state each toggle independently.

Toggles

  • Subcopy — present (why / provenance) vs. omitted (compact).
  • Pills — none · 1–2 AI actions · program link.
  • State — default · in-progress (tally) · done (green check).

Compact rule

Drop subcopy when it only restates the heading (“Eat protein” + “across your meals”). Keep it when it carries the why or where it came from.

5 · Interaction & motion

How completion feels, and the two ways to log a countable habit. Tap the Done buttons and steppers below.

“Done” animation

  • Check circle pops in with a spring — scale 0.5 → 1.22 → 1 (340ms).
  • A sparkle burst — 7 stars radiate out and fade (620ms).
  • The “Done” label collapses away, leaving just the green check chip (350ms).
  • A green ring ripples out + the card gives a soft green flash + 1.5% scale-up.

Hover states

  • Card lifts (deeper shadow); Done button darkens its border; steppers & pills tint purple and nudge up 1px.

Logging is two steps (countable habits)

  • Tap the card (or its progress chip) → a bottom drawer slides up.
  • In the drawer, either dial a specific amount — a big – / + stepper plus food quick-adds (Eggs +20g, Chicken +25g…) — or hit Mark as done to complete it outright.
  • Reaching the target celebrates with the same check + sparkle as any other card.
Try it. Tap the protein or water cards on the left to open the drawer.

7 · Item schema & control resolution

// one item object → one card
{
  i: '🍗',                  // icon glyph (required)
  img: trainerPhoto,         // optional: photo fills the icon circle (fitness class → trainer headshot)
  t: 'Eat 100g of protein', // heading (required)
  src: 'Protects your muscle', // subcopy (optional)

  type: 'honor',   // 'honor' | 'device' | 'tally'
  kind: 'task',    // 'task' | 'med'  (overrides type)
  done: false,
  dur: '22 min',        // fitness class → video-duration play pill (replaces Done)

  cta: 'Start',             // task CTA label (keep short)
  ai:  'Where is my...',     // AI context
  aiLabel: 'Ask Bold',       // single-pill label

  n: 3, target: 6,          // tally: current / goal
  step: 10, unit: 'g',      // tally step + unit (water: step 1, no unit)
  n: '12 min today',       // device badge text

  pills: [{ c: 'Foods high in protein', a: '…' }]
}

Trailing control — resolution order

  • type:'device' → synced badge (read-only)
  • type:'tally' → stepper
  • kind:'task' → CTA, or static ✓ when done
  • kind:'med' → Done → opens side-effect check
  • dur present → video-duration play pill (fitness class)
  • else → honor toggle (Done on/off)

Quick actions (AI entry points)

  • pills:[…] → renders each as a quick action; tapping opens Bold with that context (or a program link).
  • else if ai present (non-task) → single quick action labelled aiLabel (e.g. “Easy protein ideas”).

8 · Copy guidelines

  • Heading — verb-first, specific, one line. “Eat 100g of protein”, not “Protein intake”.
  • Subcopy — the why or where it came from; never repeat the heading. ≤ ~6 words.
  • CTA label — action + › (“Find a location ›”).
  • Numbers come from the care plan. Protein target = 1.4–1.6 g/kg of the member’s target weight (140 lb → ~100 g/day). Keep card numbers consistent with the Care → Nutrition directive.
  • Consistent glyphs — protein 🍗 · water 💧 · movement 💪 / 🧘 · medication 💊 · labs 🩸.
